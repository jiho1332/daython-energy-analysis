## 1. 프로젝트 방향 추천

우리가 가진 데이터는 **설비별 3상 전력계측값**이다. 서버 CPU 사용률, 냉각 온도, PUE 같은 지표는 없다. 따라서 방향은 전부 "전력 파형에서 읽어낼 수 있는 것"에서 출발한다.

### 방향 A ★ 추천 — 설비별 이상·비효율 구간 자동 탐지
> 실무 적용성 높고 예선 산출물(result.csv)과 직결

**핵심 아이디어**
- 설비별로 정상 운전 패턴을 학습 → 벗어나는 구간을 이상/비효율로 자동 레이블링
- 규칙 기반(공회전, 기저부하 초과, 저역률, 3상 불균형) + ML(IsolationForest) 하이브리드 → 설명 가능성 확보
- "왜 비효율인지"를 사람이 읽을 수 있는 라벨로 출력 (예: `공회전_의심`, `역률저하`, `피크초과`)

**모델 조합 예시**
```
[1단계] 규칙 기반 탐지  → 도메인 지식으로 명확한 비효율(공회전/저역률) 즉시 플래그
[2단계] IsolationForest → 규칙으로 못 잡는 다변량 이상치 보완 (설비별 개별 학습)
[3단계] 라벨 통합       → 비효율 유형별 사람이 읽는 라벨 부여
```

**기대 효과**
- 공회전·경부하 낭비 구간 자동 식별 → 운전 스케줄 개선 근거
- 역률 저하 구간 탐지 → 역률 개선(콘덴서 등)으로 기본요금 절감 근거

---

### 방향 B ★ 추천 — 설비별 전력 수요 예측 + 피크 부하 이동
> 절감액·탄소저감을 정량화할 수 있는 방향

**핵심 아이디어**
- 설비별/시간대별 전력 수요를 예측 → 피크 시간대 부하를 비피크로 이동(load shifting) 시뮬레이션
- 배치성 설비(건조기 등)는 심야 저부하 시간대로 스케줄 이동 가능성 검토
- 계시별 요금제(경부하/중간/최대부하) 기준으로 KRW 절감 + kg CO₂ 저감 산출

**모델 조합 예시**
```
[예측] RandomForest / LightGBM (lag 특성 기반) 또는 LSTM
[최적화] 피크→비피크 부하 이동 시뮬레이션
[평가] 이동 전후 피크 저감량, 요금 절감액, 탄소 저감량
```

> **주의**: lag-1 특성이 지배적이면 R²가 비현실적으로 높게 나옴(5초 샘플링 특성). 리포트에 이 한계 명시할 것.

---

### 방향 C — 역률·3상 불균형 기반 전력 품질 진단
> 3상 계측 데이터의 강점을 살리는 차별화 주제

**핵심 아이디어**
- R/S/T 상별 전압·전류·역률 데이터로 **3상 불균형률** 계산 → 불균형 심한 설비 진단
- 저역률 구간의 무효전력 손실 정량화 → 역률 개선 시 절감 잠재량 추정
- 분전반(좌/우측 분전반) 단위로 계통 부하 배분 상태 분석

**활용 지표**
- 전류 불균형률, 전압 불균형률(NEMA 정의)
- 역률별 무효전력(reactivePowerLagging) 손실
- 상별 부하 편중도

---

### 방향 D — 탄소 저감 관점의 운전 스케줄 최적화
> ESG/지속가능성 관점, 본선 발표에 어필

**핵심 아이디어**
- 시간대별 전력 탄소배출계수(kg CO₂/kWh)를 연계 → 탄소 집약도 낮은 시간대로 배치작업 이동
- 누적에너지(accumActiveEnergy) 기반 설비별 탄소 발자국 산정
- 절감 시나리오별 연간 탄소 저감량 비교

**데이터 연계**
- 한국전력거래소(KPX) 실시간 발전믹스 → 시간대별 배출계수
- 국가 온실가스 배출계수(전력 간접배출계수) 기본값 활용

---

### 방향 E — 통합 에너지 관리 대시보드 + 이상 알림
> 방향 A+B 결과물을 현장에서 쓰게 만드는 MLOps화 (본선 솔루션용)

```
[전력계측 DB] → [예측/탐지 모델] → [설비별 KPI 산출] → [알림/대시보드]
```

- 설비별 실시간 비효율 알림
- 일간/주간 에너지·탄소 리포트 자동 생성

## 2. 파생변수 설계 아이디어

> 컬럼 기준: `activePower`, `voltageR/S/T`, `voltageRS/ST/TR`, `currentR/S/T`, `powerFactorR/S/T`, `reactivePowerLagging`, `accumActiveEnergy`, `operation`, `ts`

### 2-1. 시간 기반 파생변수

| 변수명 | 계산 방법 | 활용 목적 |
|--------|-----------|-----------|
| `hour_of_day` | 0~23 정수 | 시간대별 패턴 학습 |
| `hour_sin` / `hour_cos` | sin(2π×hour/24), cos(2π×hour/24) | 주기성 인코딩(연속성 보존) |
| `dow_sin` / `dow_cos` | sin/cos(2π×dow/7) | 요일 주기성 |
| `is_business_hours` | 가동시간대 → 1 (실데이터로 확정 필요) | 주간/야간 운전 구분 |
| `is_weekend` | 토/일 → 1 | 주말 조업 여부 반영 |
| `shift_id` | 교대 근무조 구분 (1교대/2교대/연속) | 설비별 운전 프로파일 분류 |

### 2-2. 전력 효율 파생변수

| 변수명 | 계산 방법 | 의미 |
|--------|-----------|------|
| `load_factor` | 현재 activePower / 설비 최대전력 | 부하율 (낮으면 경부하 비효율) |
| `base_load_deviation` | 현재 전력 − 비가동시간 평균전력 | 기저부하 대비 초과 부하 |
| `idle_running_flag` | operation=가동 & activePower ≈ 무부하 수준 → 1 | **공회전 탐지 (핵심)** |
| `power_delta_1min` | P(t) − P(t−1min) | 급격한 부하 변화 |
| `rolling_mean_1h` | 1시간 이동평균 | 트렌드 추출 |
| `rolling_std_1h` | 1시간 이동 표준편차 | 변동성 측정 |
| `power_volatility` | rolling_std / rolling_mean | 전력 불안정 정도 |
| `peak_load_ratio` | 현재 전력 / 역대 최대전력 | 피크 근접도 |
| `energy_per_hour` | accumActiveEnergy 시간당 증분 | 실제 소비 에너지 검증 |

### 2-3. 전력 품질 파생변수 (3상 데이터 강점)

| 변수명 | 계산 방법 | 의미 |
|--------|-----------|------|
| `power_factor_avg` | (pfR + pfS + pfT) / 3 | 평균 역률 (낮으면 손실↑, 기본요금↑) |
| `low_pf_flag` | 평균 역률 < 0.9 & 부하 존재 → 1 | 저역률 구간 탐지 |
| `reactive_ratio` | reactivePowerLagging / activePower | 무효전력 비중 |
| `current_imbalance` | (max−min 상전류) / avg 상전류 × 100 | 전류 3상 불균형률(%) |
| `voltage_imbalance` | (max편차) / avg 상전압 × 100 (NEMA) | 전압 불균형률(%) |
| `phase_load_skew` | 상별 전류 표준편차 / 평균 | 상 부하 편중도 |

### 2-4. 이상 탐지용 통계 파생변수

| 변수명 | 계산 방법 | 의미 |
|--------|-----------|------|
| `zscore_power` | (P(t) − μ_rolling) / σ_rolling | 이상 전력 점수 |
| `iqr_flag` | Q1−1.5×IQR / Q3+1.5×IQR 위반 여부 | 통계적 이상치 플래그 |
| `lag_features` | P(t−1), P(t−2), … | 자기상관 특성 (예측용) |
| `diff_1h` / `diff_24h` | P(t) − P(t−1h/24h) | 단기/일간 변화량 |
| `residual_from_forecast` | 실제값 − 예측값 | 예측 잔차 기반 이상 |

### 2-5. 탄소 파생변수

| 변수명 | 계산 방법 | 의미 |
|--------|-----------|------|
| `carbon_intensity` | 시간대별 배출계수 (kg CO₂/kWh) | 그리드 탄소 집약도 |
| `carbon_emission_hourly` | 시간당 에너지(kWh) × 배출계수 | 시간당 탄소 배출량 |
| `low_carbon_window` | 배출계수 < 임계값 → 1 | 저탄소 배치작업 추천 구간 |
| `carbon_per_equipment` | 설비별 누적에너지 × 배출계수 | 설비별 탄소 발자국 |

### 2-6. 복합 파생변수

| 변수명 | 계산 방법 | 의미 |
|--------|-----------|------|
| `efficiency_score` | load_factor × power_factor_avg | 종합 운전 효율 점수 |
| `waste_index` | idle_running_flag 가중 + base_load_deviation | 비효율 종합 지수 |
| `savings_potential` | (현재 − 최적) 전력 × 요금단가 | 설비별 절감 잠재량(KRW) |

## 3. 핵심 도메인 지식

### 3-1. 공장 전력 계통 구조

```
[전력 공급]
  한전 수전 → 수변전 설비(변압기) → 분전반(좌/우측) → 개별 설비
                                          ↓
                            분쇄기 · 건조기 · 호이스트 · 각 호기 등
```

- **분전반(좌측/우측)**: 여러 설비로 전력을 분배하는 계통 노드 → 계통 단위 부하 분석에 활용
- **개별 설비**: 분쇄기, 건조기, 호이스트, 각 호기(공정 설비), 전등(L-1) 등

### 3-2. 3상 전력의 기본 관계

**전압 체계 (국내 저압 표준)**
- 상전압(voltageR/S/T): 약 220V (각 상 ↔ 중성선)
- 선간전압(voltageRS/ST/TR): 약 380V (상 ↔ 상, = 220 × √3)

**전력 삼각형**
```
피상전력(S, VA)² = 유효전력(P, W)² + 무효전력(Q, VAR)²
역률(PF) = 유효전력 / 피상전력 = cosθ
```
- 유효전력(activePower): 실제 일하는 전력
- 무효전력(reactivePowerLagging): 유도성 부하(모터 등)에서 발생, 실제 일은 안 하지만 계통 부담
- 역률이 낮으면 → 같은 일을 하는 데 더 큰 전류 필요 → 손실 증가 + **기본요금 할증**

### 3-3. 제조설비 운전 프로파일 (전형적 패턴)

| 프로파일 | 특징 | 예시 설비 |
|----------|------|-----------|
| **주간조 전용** | 업무시간에만 가동, 야간 정지 | 호이스트, 일부 호기 |
| **연속 운전** | 24시간 가동, 부하 변동 작음 | 분전반 기저부하, 상시 설비 |
| **배치 운전** | 특정 시간대 집중 가동, on/off 뚜렷 | 건조기, 분쇄기 |

> 실제 가동시간·교대패턴은 **실데이터로 반드시 확정**해야 함 (현재 가정값 사용 중)

### 3-4. 주요 비효율 유형과 판단 근거

**공회전(Idle Running)**
- 설비가 "가동" 상태인데 실제 유효전력이 무부하 수준에 머무름
- 생산은 없이 전력만 소비 → 대표적 낭비
- 판단: `operation=가동 AND activePower ≈ 무부하 기준선`

**경부하 운전(Low Load Factor)**
- 설비 정격 대비 매우 낮은 부하로 장시간 운전
- 효율이 정격 부하점보다 떨어지는 구간

**저역률(Low Power Factor)**
- 역률 0.9 미만 → 무효전력 비중 과다 → 기본요금 할증 대상
- 역률 개선용 콘덴서(진상용) 설치로 개선 가능

**3상 불균형(Phase Imbalance)**
- 상별 전류/전압 편차 과다 → 특정 상 과열, 손실, 설비 수명 단축
- NEMA 기준 전압 불균형 2% 초과 시 주의

**피크 부하(Peak Load)**
- 특정 시간대 부하 집중 → 최대수요전력 상승 → 기본요금(피크요금) 상승
- 배치 이동 가능 부하를 비피크로 옮기면 절감

### 3-5. 공장 전력 피크 패턴

- **주간 피크**: 조업 시간대(교대 근무 집중 시간)
- **배치 피크**: 배치 설비(건조 등) 동시 가동 시
- **계절 피크**: 여름/겨울 냉난방·공조 부하 겹칠 때
- **동시 가동 피크**: 여러 설비가 우연히 겹쳐 가동될 때

### 3-6. AI/ML 적용 시 주의사항

**데이터 품질**
- 센서 결측/오류 처리 → 이상탐지 노이즈 유발
- 5초 고빈도 샘플링 → lag-1 자기상관 매우 강함 → 예측 R² 과대평가 위험 (**리포트 명시**)
- 라벨 부재 → 비지도/규칙 기반 중심

**모델 선택 기준**
- 설명 가능성 필요 → 규칙 기반 + Tree 계열
- 실시간 탐지 → 경량 모델(IsolationForest)
- 비지도 이상탐지의 낮은 정밀도(precision)는 정상 → 리포트에서 프레이밍 중요

**설비별 개별 학습 원칙**
- 설비마다 운전 프로파일이 완전히 다름 → 전체를 한 모델로 학습하면 안 됨
- 설비별 개별 모델 또는 설비 특성을 변수로 반영

## 4. 참고 문헌 및 출처

### 4-1. 제조·산업 에너지 관리

- ISO 50001. *Energy management systems — Requirements with guidance for use.*
  → 산업 에너지 경영시스템 국제 표준, EnMS 프레임워크
- IEC 60034-30-1. *Efficiency classes of line operated AC motors (IE code).*
